1) Абстракция - сначала определяме интерфейсы, абстрактные классы, родителей. Абстрактные методы в этих классах
    а потом в конкретном классе пишем реализацию
    class Tst extends Cl
    Абстрактный класс может содержать:
        реализацию методв,
        свойства,
        конструкторы
    Обычный класс наследует абстрактный класс
    Интерфейсы могут содержать:
        методы без реализации
        методы с реализацией но отмеченные default,
        статические свойста
        !если метод static то он обязательно должен содержать реализацию

2) Инкапсуляция - ограничение доступа к данным и возможностям их изменения
    В языке реализуется благодаря модификаторам доступа

    Модификаторы доступа
        private - область видимости текущий в классе
        protected - обращаться внутри текущего класса, в наследниках, внутри паетка
        public - дает возможность обращаться откуда угодно
        default - когда модификатора нет (еще называется package private) обращаться из того же класса или пакета

3) Наследование - если какие то классы имеют набор общих характеристик и методов их нужно выносить в отдельный
    родительский класс, потом эти классы будут наследоваться от родительского и получать в наследство их свойства и поля
    (часть повторяющегося функционала - вынести в отдельный класс)
    class A extends B {}

4) Полиморфизм - это свойство системы испльзовать объекты с одинаковым интерфейсом без информации о тие и внутренней структуре объекта
