ПРИНЦИПЫ SOLID
1) S: Single Responsibility Principle - Принципы единственной ответственности
    каждый класс должен иметь строго ограниченную зону ответственности

2) O:Open-Closed Principle - Принцип открытости-закрытости
    Классы должны быть открыты для расширения (добавление нового функционала),
    но закрыты для модификации (изменение кода существующих классов)

3) L:Liskov Subsituation Principle - Принцип подстановки
    Тоже самое что полиморфизм наследование

4) I: Interface Segregation Principle - Принцип разделения интерфейса
    Интерфейсы не должны обязывать классы реализовывать методы, которые им не нужны
    В таком случае лучше создать несколько интерфейсов,чем один
    class C implements SuperInterFace {}

5) D: Dependency Inversion Principle - Принцип инверсии зависимостей
    Модули верхних уровней не должны зависеть от модулей нихних уровней.
    Оба типа модулей должны зависеть от абстракицй.

    Абстракции не должны зависеть от деталей. Детали должны зависеть от абстракиций
    Вначале должно идти проектирование:
    - интерфейсов,
    - потом абстрактных классов
    - потом их конкретных реализаций
