В Java у каждого примитива есть объектный аналог:

Примитив        Обертка
byte            Byte
short           Short
int             Integer
long            Long
float           Float
double          Double
char            Character
boolean         Boolean

Разница:
    примитив — просто значение
    обёртка — объект (лежит в куче, наследуется от Object)
    int a = 5;
    Integer b = 5; // объект

2️⃣ Когда использовать обёртки вместо примитивов
✅ 1. Дженерики
    List<Integer> list = new ArrayList<>(); - ✅
    // List<int> ❌ невозможно

2. Когда значение может быть null
    Integer count = null;   ✅
    int count = null;      ❌
    Потому что:
    int — это значение
    Integer — это ссылка на объект
    null — это отсутствие ссылки, а не отсутствие значения

❌ Когда НЕ надо использовать обёртки
    в горячих циклах
    в математике
    в low-level коде

Почему:
    медленнее
    больше памяти
    есть риск NullPointerException
